🔐 String Encode/Decode Algorithm

Complete Interview Guide for 10 LPA+ Roles

💡 The Solution

Key Insight: Use length + "#" + string format to avoid ambiguity with delimiters

Complete Java Implementation:

class Solution { public String encode(List<String> strs) { StringBuilder sb = new StringBuilder(); for(String hj : strs) { sb.append(hj.length()).append("#").append(hj); } return sb.toString(); } public List<String> decode(String str) { List<String> gh = new ArrayList<>(); int i = 0; int j = 0; while(i < str.length()) { // Find the delimiter '#' while(str.charAt(j) != '#') { j++; } // Parse the length (handles multi-digit numbers) int num = Integer.parseInt(str.substring(i, j)); // Extract the string of that length gh.add(str.substring(j + 1, j + num + 1)); // Move to next encoded string i = j + num + 1; j = i; } return gh; } }
Example:
Input: ["cat", "dog", "bird"]
Encoded: 3#cat3#dog4#bird
Decoded: ["cat", "dog", "bird"]

❌ Why Simple Delimiter Fails

Problem with Brute Force Approach: Using delimiters like , or # directly fails when strings contain those characters!

❌ WRONG Approach

// Just use '#' as delimiter encode: "hello#world#test" // But what if input is: ["he#llo", "wor#ld"] // Encoded: "he#llo#wor#ld" // On decode: splits incorrectly!

✅ CORRECT Approach

// Use length + '#' + string ["he#llo", "wor#ld"] // Encoded: "6#he#llo5#wor#ld" // Decode knows: // - Read 6 chars after first '#' // - Read 5 chars after second '#' // Works perfectly!

🎯 Critical Interview Questions (10 LPA Level)

HARD

Q1: What if the string length is very large (multi-digit)?

Example: Encoded string: 12#helloworld!!
Your decode logic must correctly read ALL digits before '#'

✅ CORRECT

while(str.charAt(j) != '#') { j++; } int length = Integer.parseInt( str.substring(i, j) ); // Handles: 1, 10, 100, 1000+

❌ WRONG

char ch = str.charAt(j-1); int length = ch - '0'; // For "10#abcdefghij" // Gets only '0', not 10! // Fails for numbers > 9
MEDIUM

Q2: Why is this solution O(n)? Prove it.

Interviewers ask this to check complexity understanding.

Operation Time Complexity Explanation
Encode O(n) Iterate over every character once
Decode O(n) Scan each character once while moving pointers
Space O(n) Store the encoded/decoded result
Why it's optimal: We MUST read every character at least once, so O(n) is the best possible complexity.
MEDIUM

Q3: Why use StringBuilder instead of String concatenation?

❌ Bad Code (O(n²))

String result = ""; for(String word : strs) { result += word; // Creates new object! } // Each += creates new String object // Time: O(n²)

✅ Good Code (O(n))

StringBuilder sb = new StringBuilder(); for(String word : strs) { sb.append(word); // Modifies existing } // StringBuilder is mutable // Time: O(n)
Interview Answer: StringBuilder is mutable and avoids creating multiple intermediate string objects, making concatenation efficient from O(n²) to O(n).
EASY

Q4: What happens if input contains an empty string?

Test Case:
Input: ["", "hello", ""]
Encoded: 0#5#hello0#
Decoded: ["", "hello", ""]
Why it works:
• Length = 0
• substring(j+1, j+0+1) = substring(start, start)
• Returns empty string correctly!
HARD

Q5: Can you handle strings with special characters or Unicode?

Test Case:
Input: ["hello🎉", "नमस्ते", "test#123"]
Encoded: 8#hello🎉18#नमस्ते8#test#123
Watch Out: String.length() in Java counts UTF-16 code units, not actual characters!
Emoji like 🎉 might count as 2 units.
// For production code, consider: str.codePointCount(0, str.length()) // Actual character count
MEDIUM

Q6: How would you modify this to handle null strings?

public String encode(List<String> strs) { if(strs == null) return ""; StringBuilder sb = new StringBuilder(); for(String str : strs) { if(str == null) { sb.append("-1#"); // Special marker for null } else { sb.append(str.length()).append("#").append(str); } } return sb.toString(); } public List<String> decode(String str) { if(str == null || str.isEmpty()) return new ArrayList<>(); List<String> result = new ArrayList<>(); int i = 0, j = 0; while(i < str.length()) { while(str.charAt(j) != '#') j++; int len = Integer.parseInt(str.substring(i, j)); if(len == -1) { result.add(null); // Restore null } else { result.add(str.substring(j+1, j+len+1)); } i = j + len + 1; j = i; } return result; }
HARD

📋 Mini Cheat Sheet for Interviews

Key Idea:

length + "#" + string

Example Walkthrough:

Input: ["cat", "dog"]

Encoding Steps:
  1. "cat" has length 3 → append "3#cat"
  2. "dog" has length 3 → append "3#dog"
  3. Result: "3#cat3#dog"

Decoding Steps:
  1. Find '#' at position 1
  2. Read number before '#' → 3
  3. Extract 3 characters after '#' → "cat"
  4. Move pointer to position 5
  5. Repeat for next string → "dog"
Pointer Movement Visualization: String: "3#cat3#dog" ↑ ↑ ↑ ↑ i j i j Step 1: i=0, j=0 Find '#' → j=1 Length = "3" Extract: "cat" Move: i=5, j=5 Step 2: i=5, j=5 Find '#' → j=6 Length = "3" Extract: "dog" Done!

⚡ Complexity Analysis

Aspect Complexity Reason
Encode - Time O(n) Single pass through all characters
Encode - Space O(n) StringBuilder stores result
Decode - Time O(n) Single pass with two pointers
Decode - Space O(n) ArrayList stores result
Where n = total number of characters in encoded string
This is optimal because we must read every character at least once.

🎓 Interview Tips for 10 LPA Roles

What to Say During Interview:

  1. Start with brute force: "First thought is to use a delimiter like comma, but that fails when strings contain that character."
  2. Present the solution: "Better approach is to prefix each string with its length, so we know exactly how many characters to read."
  3. Handle edge cases: "This works even for empty strings, multi-digit lengths, and special characters."
  4. Discuss complexity: "Both encode and decode are O(n) time and space, which is optimal."
  5. Mention optimizations: "I used StringBuilder for O(n) concatenation instead of String which would be O(n²)."

Common Mistakes to Avoid:

  • ❌ Using ch - '0' for multi-digit lengths
  • ❌ Using String concatenation instead of StringBuilder
  • ❌ Not handling empty strings
  • ❌ Forgetting to update both pointers (i and j)
  • ❌ Not testing with strings containing '#' character
