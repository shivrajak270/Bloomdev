Valid Anagram

// 3 approaches · complexity tables · visual intuition

APPROACH 01
StringBuffer Delete O(n²) T O(n) S
Idea: Copy s into a StringBuffer. For each char in t, find & delete it. If any char is missing → not anagram. 
if (s.length() != t.length()) return false;

StringBuffer sb = new StringBuffer(s);   // O(n)

for (int i = 0; i < t.length(); i++) {
    char ch = t.charAt(i);             // O(1)
    int  n  = sb.indexOf(String.valueOf(ch)); // O(n) ←bottleneck
    if (n == -1) return false;
    sb.deleteCharAt(n);               // O(n) shift!
}
return true;

APPROACH 02
Sort & Compare O(n log n) T O(n) S
Idea: Convert both strings to char arrays, sort them. Equal sorted arrays = anagram. 
if (s.length() != t.length()) return false;

char[] ch = s.toCharArray();   // O(n)
char[] jk = t.toCharArray();   // O(n)

Arrays.sort(ch);               // O(n log n)
Arrays.sort(jk);               // O(n log n)

return Arrays.equals(ch, jk);  // O(n)
Visual:

t = "nagaram" → sorted → a a a g m n r ✓

APPROACH 03 ⭐ OPTIMAL
Frequency Array O(n) T O(1) S
Idea: One int[256] array. Increment for chars in s, decrement for chars in t. If all zeros → anagram. 
if (s.length() != t.length()) return false;

int[] ch = new int[256];      // O(1) space (fixed size)

for (int i = 0; i < s.length(); i++) {
    ch[s.charAt(i) - 'a']++;   // count s chars
    ch[t.charAt(i) - 'a']--;   // cancel t chars
}

for (int y : ch)
    if (y != 0) return false; // leftover → mismatch

return true;
Visual for "rat" vs "tar":

s='r' → ch['r'-'a']++ → ch[17]=1
t='t' → ch['t'-'a']-- → ch[19]=-1
s='a' → ch['a'-'a']++ → ch[0]=1
t='a' → ch['a'-'a']-- → ch[0]=0 ✓
s='t' → ch['t'-'a']++ → ch[19]=0 ✓
t='r' → ch['r'-'a']-- → ch[17]=0 ✓
All zeros → true ✓

💡 Bonus interview Q: Why is String immutable but StringBuilder/StringBuffer mutable?
→ String is stored in the String Pool; JVM reuses identical literals for memory efficiency & thread safety. StringBuilder uses a mutable char[] internally — no pool, no reuse, direct mutation. StringBuffer adds synchronized = thread-safe but slower.
