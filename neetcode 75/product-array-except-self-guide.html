🎯 Product of Array Except Self

Complete Interview Guide for 10 LPA+ SDE Roles

All Three Solutions Explained

1️⃣ Brute Force Solution

Time: O(n²)
Space: O(1)

Approach: For each index, multiply all elements except itself using nested loops.

class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] ans = new int[n];
        
        for(int i = 0; i < n; i++) {
            int sum = 1;
            for(int j = 0; j < n; j++) {
                if(i == j)
                    continue;
                sum *= nums[j];
            }
            ans[i] = sum;
        }
        return ans;
    }
}
💡 Interview Tip: Always start with this approach to show structured thinking!

2️⃣ Better Solution (Prefix & Suffix Arrays)

Time: O(n)
Space: O(n)

Approach: Create separate prefix and suffix arrays, then multiply them.

class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] ans = new int[n];
        int[] left = new int[n];
        int[] right = new int[n];
        
        left[0] = 1;
        right[n-1] = 1;

        // Build left array
        for(int i = 1; i < n; i++) {
            left[i] = left[i-1] * nums[i-1];
        }

        // Build right array
        for(int i = n-2; i >= 0; i--) {
            right[i] = right[i+1] * nums[i+1];
        }
        
        // Multiply left and right
        for(int i = 0; i < n; i++) {
            ans[i] = left[i] * right[i];
        }
        return ans;
    }
}
💡 Key Insight: For each index i, result = (product of all left elements) × (product of all right elements)

3️⃣ Most Optimal Solution (Single Array)

Time: O(n)
Space: O(1)

Approach: Use the answer array itself to store prefix, then multiply with suffix.

class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int prefix = 1;
        int suffix = 1;
        int[] ans = new int[n];

        // First pass: store prefix products
        for(int i = 0; i < n; i++) {
            if(i == 0)
                ans[i] = 1;
            else {
                prefix = prefix * nums[i-1];
                ans[i] = prefix;
            }
        }
        
        // Second pass: multiply with suffix products
        for(int i = n-1; i >= 0; i--) {
            if(i == n-1)
                ans[i] *= 1;
            else {
                suffix = suffix * nums[i+1];
                ans[i] = ans[i] * suffix;
            }
        }
        return ans;
    }
}
⭐ This is the solution interviewers expect at 10 LPA+ level!

Interview Preparation Guide

❓ Question 1: What's the brute force approach?

Your Answer: "For each index i, I'll multiply all elements except i using nested loops. Time complexity is O(n²)."

❓ Question 2: Can you optimize it?

Your Answer: "Yes! For every index i, the result equals the product of elements to the left multiplied by the product of elements to the right. I can precompute these using prefix and suffix products in O(n) time."

❓ Question 3: Why can't we use division?

Your Answer: "Because the array might contain zeros. For example, [1,2,0,4] would give a total product of 0, making division fail. Also, multiple zeros would make the logic complex."

❓ Question 4: What if there's one zero?

Example: Input: [1, 2, 0, 4]

Output: [0, 0, 8, 0]

Explanation: Only the index containing zero gets the product of remaining numbers. All other indices will be 0.

❓ Question 5: What if there are two zeros?

Example: Input: [1, 0, 3, 0]

Output: [0, 0, 0, 0]

Explanation: Every index will have at least one zero in its product, resulting in all zeros.

❓ Question 6: Can you do this without extra arrays?

Your Answer: "Yes! I can use the answer array itself to store prefix products first, then multiply with suffix products in a second pass. This gives O(1) space complexity (excluding output)."

Edge Cases to Mention

  • Array size = 1 → return [1]
  • Array contains zeros (one or multiple)
  • Array contains negative numbers
  • Large numbers (potential overflow)

Common Mistakes to Avoid

  • Using division approach
  • Using nested loops in optimal solution
  • Forgetting to initialize prefix = 1, suffix = 1
  • Incorrect order of suffix multiplication
  • Not handling zero cases

Pro Tips for 10 LPA+ Interviews

✅ Best One-Line Explanation:

"Instead of recalculating the product for each index, we precompute prefix and suffix products so each element gets the multiplication of left and right in O(n) time."

🎯 How to Structure Your Explanation

  1. State the problem: "We need to find the product of all elements except the current one."
  2. Mention constraints: "Without using division and in O(n) time."
  3. Explain the approach: "Use prefix-suffix technique."
  4. Walk through complexity: "Two passes, O(n) time, O(1) extra space."
  5. Mention edge cases: "Handle zeros and negatives."

🔥 What Makes You Stand Out

  • Immediately mention all three approaches (brute, better, optimal)
  • Connect to the prefix-suffix pattern family
  • Proactively discuss edge cases before being asked
  • Explain why division doesn't work (shows deep understanding)
  • Mention space optimization without being prompted

📝 Key Points to Remember

  • Left product: Product of all elements to the left of index i
  • Right product: Product of all elements to the right of index i
  • Result: ans[i] = left[i] × right[i]
  • Optimization: Store left in ans[], multiply right on the fly

🎓 Advanced Follow-up Questions

Q: Can you solve this for a 2D array?

A: "Yes! Apply the same prefix-suffix logic for rows and columns separately."

Q: How would you handle very large numbers?

A: "Use modulo operations or BigInteger class to prevent overflow."
